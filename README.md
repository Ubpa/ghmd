# ghmd — GitHub Markdown Preview

Renders Markdown with full GitHub feature support. Works as both a **standalone server** (browser) and a **VS Code extension**.

## Features

- GitHub Alerts (`[!NOTE]`, `[!TIP]`, `[!IMPORTANT]`, `[!WARNING]`, `[!CAUTION]`)
- Mermaid diagrams (flowchart, sequence, gantt, class, state, ER, pie, git)
- LaTeX math via KaTeX (inline `$...$` and block `$$...$$`)
- `<details>` collapsible sections
- Footnotes, task lists, diff highlighting
- `<kbd>`, `<sub>/<sup>`, `<picture>` theme-aware images
- Code syntax highlighting (highlight.js)
- Light/dark theme toggle
- Live reload on file change

## Standalone Server

```bash
npm install
node serve.mjs README.md           # open http://localhost:6419
node serve.mjs README.md 8080      # custom port
```

KaTeX and Mermaid load from CDN by default. For fully offline use:

```bash
node serve.mjs --init              # downloads katex + mermaid locally
node serve.mjs README.md           # now works without network
```

## VS Code Extension

### Install from .vsix

```bash
npm run package                    # creates ghmd-0.1.0.vsix
code --install-extension ghmd-0.1.0.vsix
```

### Usage

1. Open any `.md` file
2. `Cmd+Shift+V` (Mac) / `Ctrl+Shift+V` (Win/Linux)
3. Or Command Palette: `GHMD: Open Preview to the Side`

### Development

```bash
npm install                        # install deps
npm run build                      # bundle extension + vendor assets
npm run dev                        # build with sourcemaps (for F5 debugging)
npm run package                    # build + create .vsix
```

## Project Structure

```
ghmd/
  serve.mjs             Standalone server (single file, zero build step)
  src/extension.cjs     VS Code extension source
  scripts/vendor.mjs    Copies dist files from node_modules -> vendor/
  dist/extension.js     Bundled extension output (generated by esbuild)
  vendor/               Runtime assets for the VS Code extension:
    css/                github-markdown-css + highlight.js themes
    katex/              katex.min.js, katex.min.css, fonts/
    mermaid.min.js      Mermaid diagram renderer
```

## How It Works

Both the server and extension use the same rendering pipeline:

1. **marked** parses Markdown (GFM) with plugins for alerts and footnotes
2. **highlight.js** does code syntax highlighting
3. **github-markdown-css** provides GitHub's exact styling
4. **KaTeX** (CDN or local) renders LaTeX math client-side
5. **Mermaid** (CDN or local) renders diagrams client-side

The VS Code extension bundles marked/hljs into `dist/extension.js` via esbuild, and ships only the needed KaTeX/Mermaid dist files in `vendor/` (~3.6 MB uncompressed, ~1.5 MB in .vsix).

## Updating Dependencies

```bash
npm update                         # update node_modules
npm run build                      # rebuild vendor/ + dist/
npm run package                    # create new .vsix
```
